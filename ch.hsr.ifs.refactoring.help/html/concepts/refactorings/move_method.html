Move Method

Description

The Move Method refactoring transfers a method of a class to another. This is useful if there is a method in a class that is not really responsible for the functionality provided by this method. If desired a delegate method can be left in place of the old method if the interface of the source class should be left untouched. All calls will be delegated to the method's new place.

Conditions

  • The caret has to be set at the method definition which should be moved.
    •

    Options


    To move a method to another class, there has to be at least one other class visible to the current fields class. And an instance of this class has to be created in the source classes constructor because the references have to be updated. The possible classes are listed in the dialog:

    Option Description
    Move to class Here the target class can be selected among the possible target classes to move the method to.
    Leave a delegate... For not changing the interface of the source class a delegate method can be left in place of the moved method. It will foreward the calls.
