Move Field

Description

This refactoring moves, as its name says, a field from one class into another one. This is useful if the responsibility for a field is not in the owning class, but in another one. In this case, the field should better be moved to that class.
References to that field will be updated and point then to the new field.

Conditions

  • The caret has to be set at the field which should be moved.
    •

    Options


    To move a field to another class, there has to be at least one other class visible to the current fields class. And an instance of this class has to be accessible through another field of the current class because the references have to be updated. The possible classes and fields are listed in the dialog:

    Option Description
    Move to class Here the target class can be selected among the possible target classes to move the field to.
    Class is accessible by The field which can be selected here has to point at an instance of the class seleced as target.
